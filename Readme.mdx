Este projeto é um monorepo usando o [lerna](https://lernajs.io/). Cada pacote dentro de _/packages_ é um projeto individual e são publicados individualmente para o [npm](https://www.npmjs.com/org/pismo).

## Usando o **Bolt** em uma aplicação

- Importe o provider do bolt e adicione no topo da aplicação
- Importe os componentes que precisar utilizando o padrão _@pismo/bolt-\[component-name\]_

```
// App.js
import {Bolt} from '@pismo/bolt-core'
import {TextField} from '@pismo/bolt-text-field'
import Button from '@material-ui/core/Button'


<Bolt>
    <TextField placeholder='e-mail' />
    <Button>Send</Button>
</Bolt>
```

> Para maiores informações, consulte a documentação de cada pacote

## Desenvolvimento

Para criar um novo componente siga os passos abaixo:

- Clone este _monorepo_
- Crie um novo _branch_ ex: feature/my-Button
- Instale as dependências com o comando `yarn`
- Crie o componente usando o comando `yarn template:component [component]`
  (substituindo \[component\] pelo nome do seu componente. ex `yarn template:component MyButton`)
- Crie um arquivo de documentação ex: MyButton.mdx
- Crie testes unitários na pasta \_\_test\_\_
- Faça o commit e submeta um PR (pull request)

## Documentação

Para a documentação usamos a biblioteca [Docz](https://www.docz.site/)

Para incluir uma nova página basta criar um documento com a extensão _.mdx_

ex:

```
// MyButton.mdx
___
'name': MyButton
___

import MyButton from './MyButton'
import { Playground } from 'docz'

# Titulo do Meu documento

## Qualquer coisa como markdown

<p style={{textAlign: 'center'}}>Também aceita jsx elementos</p>

<Playground>

ou

<Playground>
    {
        () => {
            const handlerClick = (e) => {

            }

        }
    }
```

Para rodar a documentação no modo desenvolvimento use o comando `yarn docs:dev`

Para buildar a documentação use o comando `yarn docs:build`

## Formatação

Usamos o padrão _[Standard](https://standardjs.com/)_ de formatação junto com o _Prettier_

Por padrão sempre que é feito o commit, automaticamente roda o comando para formatação, porém se quiser fazer a formatação
ao longo do desenvolvimento pode executar o comando `yarn format`

## Publicação

Para publicar os pacotes no _npm_ é necessário primeiro ser um team member da org @pismo:developers.

- Faça login no npm usando `npm login`
- Rode o comando `yarn lerna:publish`
- Você será questionado sobre a próxima versão que estará publicando
- Rode o comando `npm run manual-publish` **este comando deve ser rodado com npm como descrito**
